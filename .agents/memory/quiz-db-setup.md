## Rule
The quiz_questions and quiz_results tables are defined in lib/db/src/schema/quiz.ts but are NOT auto-migrated on server start. They must be pushed manually.

## How to apply
1. `pnpm --filter @workspace/db push` — creates/updates all tables via drizzle-kit push
2. `node lib/db/seed-quiz.mjs` — seeds 70 questions (10 per chapter, 7 chapters)
3. Restart the api-server workflow after schema changes

**Why:** The db package uses drizzle-kit push (not auto-migrations) and there is no seed step in the startup workflow. First-run environments will have empty tables and the quiz will show "walang tanong" (no questions) until seeded.

## Error analysis
The whyWrong field in quiz submission is generated by generateWhyWrong() in artifacts/api-server/src/routes/quiz.ts. It uses pattern matching on the question/answer text to identify specific misconceptions (formula inversions, concept confusion pairs, etc.) rather than a generic message.
